# 日報シュシュ 仕様説明書

## 1. 概要
日本語の日報入力を最短3分で完了できる、シンプルで洗練されたWebアプリです。ダッシュボード、店舗KPI、入力フォーム、AIインサイト、タイムライン、通知設定を1画面で提供します。入力内容はローカルストレージに保存され、再訪時に自動復元されます。

## 2. 目的
- 日報入力の負担を軽減し、継続しやすい体験を提供する。
- 入力内容を整理し、次のアクションを明確にする。
- 日本語UIに最適化したレイアウト・文言で、導入コストを最小化する。

## 3. 画面構成
### 3.1 ヘッダー
- ブランド表示（ロゴ、名称、タグライン）
- アクションボタン
  - 「テンプレート」: 拡張用のダミーUI
  - 「日報を送信」: 拡張用のダミーUI

### 3.2 ダッシュボード（ヒーローエリア）
- 今日の進捗・集中度・平均作業時間の可視化
- タグで機能の特徴を表示

### 3.3 今日のログ
- 完了タスク、連絡/相談、集中セッション、改善メモのサマリー
- 進捗バーと目標入力率

### 3.4 日報入力フォーム
- 今日の成果
- 課題・相談事項
- 明日の予定
- 操作ボタン
  - 「クリア」: 入力を初期化
  - 「日報を保存」: ローカル保存

### 3.5 AIインサイト
- 入力内容の改善提案（固定メッセージ）

### 3.6 今週のタイムライン
- 週次のメモを簡易的に表示

### 3.7 通知設定
- 通知ON/OFFの視覚的トグル

## 4. 機能要件
- ローカルストレージに入力内容を保存・復元する。
- フォーム送信時に保存し、アラートで完了を通知する。
- クリア操作でフォーム初期化および保存データ削除を行う。
- 保存のオン/オフを選択でき、オフの場合は保存しない。
- 店舗KPI（売上、来客数、ピーク時間帯、目標達成率）を入力し、サマリーを自動生成する。
- KPIや日報入力をJSONとしてダウンロードできる。
- パスコードによる簡易ロックを提供し、端末内でアクセス制御する。
- レスポンシブデザインでデスクトップ/タブレット/スマホに対応。

## 5. 非機能要件
- システムフォントを利用し、外部フォント取得を行わない
- シンプルな静的構成で動作（ビルド不要）
- 主要ブラウザでの表示崩れがないこと
- CSP（Content-Security-Policy）を適用して外部リソース読み込みを制限する

## 6. ファイル構成
```
/ (リポジトリ直下)
├── index.html  # 画面UIとレイアウト
├── styles.css  # デザイン・レスポンシブ対応
├── app.js      # 入力保存/復元ロジック
└── SPEC.md     # 本仕様書
```

## 7. 操作フロー
1. ユーザーがフォームに入力
2. 「日報を保存」を押す
3. ブラウザに入力内容を保存
4. 再訪時に入力内容を自動復元
5. 「クリア」で初期化
6. 「レポートをダウンロード」でJSONとしてエクスポート

## 8. 制限事項
- サーバー通信は未実装（ローカルのみ）
- AIインサイトは固定文言
- 送信/テンプレート機能はUIのみ
- 入力内容はブラウザ内にのみ保存されるため、端末共有時は注意が必要
- パスコードは端末内保存のため、端末共有時はログアウト（ロック）推奨

## 9. 今後の拡張案
- 日報テンプレートの選択機能
- チーム共有（API連携）
- 週次/月次レポート自動生成
- 通知のスケジュール設定

## 10. ワンアクションで公開する方法（静的ホスティング）
1. `index.html` / `styles.css` / `app.js` をZIP化
2. GitHub Pages や Netlify などの静的ホスティングにドラッグ＆ドロップでアップロード
3. 自動発行されたURLにアクセスすれば即利用可能
